# Plan: Add Auto-Add Configuration for Random Fill Tags and Fix Blacklist Auto-Load

## Problem (Original)
When adding a random fill tag, users must manually click "Add >>" to move videos from the collection to the "Added Videos" list. After saving and later editing the tag, the added videos list is empty, requiring manual re-addition. The user requests an "auto add" config option to automatically populate the added videos list.

## Problem (Additional Bug - Reported 2026-05-02)
The blacklist is auto-selected via collection profile, but the added videos count still equals the full collection count. Only after manually pressing the "Load" button (to load the blacklist file) does the count drop to reflect blacklisted videos being removed. This indicates that the auto-load of blacklist from profile does not actually filter the `added_videos` list.

## Root Cause (Original)
In `RandomFillDialog.__init__()`, when editing an existing tag, the method `load_collection()` is called but it clears `self.added_videos` and never restores the tag's previously selected videos (`tag.collection_videos`). The dialog should populate `added_videos` from the tag's `collection_videos` after loading the collection.

Additionally, manual selection is not persisted between dialog sessions unless the tag is saved.

## Root Cause (Additional Blacklist Bug)
The blacklist combo (`blacklist_profile_combo`) is populated with filenames found in multiple locations (configured blacklist path and current working directory). However, when a collection is selected, the code attempts to load the blacklist file using only the configured `blacklist_path`, ignoring the location where the file was originally discovered. Thus, if the blacklist file resides outside the configured path (e.g., in the current directory), the auto-load silently fails, leaving `self.blacklist` unset and added videos unfiltered. Manual load via the "Load" button works because the user picks the correct file.

## Solution Overview (Original)
1. **Bug Fix**: Restore `added_videos` when editing a tag by copying from `tag.collection_videos` after `load_collection()` call.
2. **Auto-Add Feature**: Add a configuration option `auto_add` (default: false). When enabled, whenever a collection is loaded (via browse or profile selection), all non-blacklisted videos are automatically added to the "Added Videos" list.

## Solution Overview (Blacklist Fix)
Store the full path of each blacklist file when populating the combo, and use that stored path for auto-loading. Update the selection handlers to retrieve the stored path via `itemData`. Fallback to the old path-construction method for compatibility.

## Changes Required (Blacklist Fix)

### 6. Store Blacklist Full Paths
In `RandomFillDialog.load_available_profiles()`:
- Replace `blacklist_files` set with a dict `blacklist_files = {}` mapping filename -> full Path.

- After placeholder, for each sorted name: `self.blacklist_profile_combo.addItem(name)` then `self.blacklist_profile_combo.setItemData(self.blacklist_profile_combo.count()-1, blacklist_files[name])`.

### 7. Update `collection_profile_selected()`
Replace the logic that constructs `bl_file = Path(blacklist_path) / bl_name` with:
```
bl_path = self.blacklist_profile_combo.itemData(i)
if bl_path and Path(bl_path).exists():
    self.load_blacklist_file(bl_path)
else:
    # fallback to old method
    _, blacklist_path_fb = get_config_paths()
    fallback = Path(blacklist_path_fb) / bl_name
    if fallback.exists():
        self.load_blacklist_file(str(fallback))
```
Note: The explicit `load_blacklist_file` call after `setCurrentIndex` can remain; it's harmless but ensures load even if signal-blocked.

## Edge Cases & Notes
- Storing first-found path ensures deterministic selection.
- Placeholder item `-- None --` should have no data; `itemData` will be None.
- Fallback logic preserves compatibility if stored data is missing.
- `load_blacklist_file` sets `self.blacklist` and filters `self.added_videos`, then refreshes UI.
- Auto-add still respects blacklist; with the blacklist fix, `self.blacklist` will be correctly set before auto-add runs (if the blacklist is auto-loaded first). However note order: `collection_profile_selected` loads collection first, then loads blacklist. If auto_add is true, auto-add runs inside `load_collection` before blacklist load. So added videos will initially include all non-blacklisted based on auto-discovery (which may be none). After blacklist loads, they get filtered. This is acceptable.
