# Plan to Remove Channel Combobox from Custom Tag

## Analysis
- The custom tag dialog is implemented by `TagDialog` class in `dialogs.py`
- `TagDialog` currently contains a channel combobox UI element (lines 142-149)
- It also has a `load_channels()` method that populates this combobox (lines 259-270)
- The method is called in `__init__` at line 66
- The Tag data model (`data_models.py`) has NO `channel` attribute, confirming this is indeed incorrect

## Changes Required

### File: dialogs.py

**1. In `TagDialog.__init__` (line 66):**
   - Remove the call: `self.load_channels()`

**2. In `TagDialog.setup_ui` (lines 142-149):**
   - Remove the entire channel_layout block:
     ```python
     channel_layout = QHBoxLayout()
     channel_layout.addWidget(QLabel("Channel:"))
     self.channel_combo = QComboBox()
     self.channel_combo.setEditable(True)
     self.load_channels()
     channel_layout.addWidget(self.channel_combo)
     channel_layout.addStretch()
     layout.addLayout(channel_layout)
     ```

**3. In `TagDialog` class:**
   - Remove the entire `load_channels()` method (lines 259-270)

## Verification
- After changes, TagDialog will no longer have a channel combobox
- Custom tags will still function correctly as Tag model doesn't store channel anyway
- RandomFillDialog (which also has a load_channels method) is unaffected as it doesn't have a channel_combo UI element and never calls load_channels()
