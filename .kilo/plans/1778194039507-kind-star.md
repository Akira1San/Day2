# Plan: Inspect Schedule Preview Feature

## Overview
Add a new button to the main window that opens a saved schedule in a separate preview window with horizontal scrolling, displaying dates and days for each day. The window will include a Cancel button to close.

---

## 1. Files to Modify

### 1.1 `daypart_scheduler.py` (Main Window)
**Changes:**
- Add "Inspect" button to bottom button layout (line ~150-197 area)
- Connect button to new `inspect_schedule()` method
- Implement `inspect_schedule()` method:
  - Get selected profile from `schedule_profile_combo`
  - Construct filename: `schedule_{profile}.json`
  - Check if file exists; show error if not
  - Load JSON and extract "calendar" (30-day) schedule data
  - Create and show `SchedulePreviewDialog` with data

### 1.2 `dialogs.py` (New Dialog)
**Changes:**
- Add new `SchedulePreviewDialog` class (QDialog):
  - Constructor accepts parent and schedule_data dict
  - UI layout:
    - Title label: "Schedule Preview - {profile}"
    - QScrollArea with horizontal scrollbar enabled
    - Inside scroll area: widget containing daily schedule entries
    - Each day shows: date (YYYY-MM-DD), day name, and entries list
    - Format: time - video name per entry
    - Cancel button at bottom to close dialog
  - Apply consistent dark theme styling (matching existing dialogs)
  - Method `load_schedule(data)` to populate display

## 3. Data Handling

### Schedule JSON Structure (existing)
```json
{
  "weekly": { ... },
  "calendar": {
    "2026-05-02_saturday": {
      "date": "2026-05-02",
      "day": "Saturday",
      "description": "Auto-generated schedule",
      "entries": [
        { "time": "00:00:00", "file": "...", "collection_id": "...", "channel": "...", "source": "random" },
        ...
      ]
    },
    ...
  }
}
```

### Loading Logic
- Read `schedule_{profile}.json`
- Extract `data["calendar"]` dict
- Sort by date key (or natural order of keys)
- Pass to dialog for display

## 5. Edge Cases & Validation

- If profile name is empty: show QMessageBox warning
- If `schedule_{profile}.json` doesn't exist: show QMessageBox "Schedule not found"
- If JSON malformed: show error message
- Dialog should be modal (setModal(True)) to prevent multiple preview windows
- Large schedules (30 days): scroll area handles efficiently; consider using QListWidget per day to avoid widget bloat
