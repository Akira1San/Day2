# Plan: Add Season-Based Sequencing for Series Tags

## Overview
Enable collection JSON files to contain metadata tags `"Series: <name>"` and `"Season: <num>"` to automatic series naming and season-based episode sequencing. When a series tag uses `play_mode = season_sequence`, episodes will be ordered by season then episode, playing all episodes of a season before moving to the next, with seasons taken from the metadata (0-indexed). Existing `sequence` behavior remains unchanged.

## Key Concepts
- Collection entries can have a `tags` array. Tags of form `"Series: X"` and `"Season: N"` are extracted.
- Videos inherit these tags from their parent collection entry.
- A new `play_mode` value `"season_sequence"` activates season-aware sequencing.
- Progression: day `N` starts at global episode index `N * video_count`. That index is mapped to a season using precomputed episode counts per season. Only episodes from the current season are placed (no cross-season mixing).

### 3. Scheduling Logic (scheduler.py & data_models.py)

#### 3.1 New Helper
**File:** `scheduler.py`  
**Method:** `_get_season_sequence_videos(tag, day_offset) -> List[dict]`

```python
def _get_season_sequence_videos(self, tag, day_offset):
    """Return list of videos to play on given day for season_sequence mode."""
    if not getattr(tag, '_has_season_tags', False):
        # Fallback to standard sequence
        return self._get_sequence_videos(tag, day_offset)

    daily_limit = getattr(tag, 'video_count', 1)
    # Global episode index (0-based)
    global_idx = day_offset * daily_limit

    # Determine which season this index falls into
    cum = 0
    for season in tag._sorted_seasons:
        season_count = tag._season_episode_counts[season]
        if global_idx < cum + season_count:
            # Found season
            local_idx = global_idx - cum
            # Max we can take from this season without crossing boundary
            remaining = season_count - local_idx
            take = min(daily_limit, remaining)
            videos = tag._season_groups[season][local_idx : local_idx + take]
            return videos
        cum += season_count

    # Offset beyond available episodes -> return empty (or wrap?)
    return []
```

## Potential Edge Cases
- Inconsistent Series tags across videos: pick most common or first.
- Missing Season tag: fallback.
- Non-integer Season values: skip.
- Episodes per season uneven: algorithm handles.
- Day truncation due to time: actual placed episodes may be fewer; the daily offset assumption may drift. Acceptable for now; document limitation.
