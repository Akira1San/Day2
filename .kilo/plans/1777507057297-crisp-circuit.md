# Plan: Implement Gap Tag Feature

## Concept
A **Gap Tag** is a special tag that automatically detects and fills time gaps between scheduled custom/series tags with interstitial content (trailers, promos, music, standby videos). It behaves like a TV channel filler, ensuring no dead air while maintaining a professional schedule flow.

## Example
Given a day with:
- Tag A: 12:00–13:06
- Tag B: 13:30–15:15

There is a 24-minute gap (13:06–13:30). A Gap Tag with a collection of 2-minute trailers would fill that gap with as many trailers as fit, possibly leaving a small remainder if exact fit isn't possible.

## Implementation Details

### 1. Data Model
- Gap Tag is a subtype of custom tag, perhaps with `is_gap_filler = True` attribute.
- It should have:
  - `collection_videos` (list of filler videos)
  - `max_duration` (optional, to limit total filler time per day)
  - `preserve_boundaries` (bool: if True, avoid placing filler that would cut into program end credits; default False)
  - `priority` (if multiple gap fillers exist, this determines which runs first)

### 2. Algorithm Placement
Gap filling should occur **after** all primary tags (custom, series, multi-series) have been placed for the day, as a final pass. It could be integrated into `_approximate_finalize_day()` or implemented as a dedicated strategy method that runs after tag placement.

Steps:
- Build a list of occupied time ranges from all placed tags (already done via `occupied_ranges`).
- Find all gaps between occupied ranges within the day (00:00–24:00). Gaps are intervals where no tag exists.
- For each gap, iterate through the Gap Tag's video collection, selecting videos to fit without overlapping occupied ranges.
- Videos can be placed sequentially; if a video doesn't fit entirely, skip it (or truncate if preserve_boundaries=False).
- Update `occupied_ranges` and `final` schedule accordingly.
- If multiple Gap Tags exist, process them in priority order, each filling remaining gaps.

### 3. Interaction with Approximate Modes
- Gap Tag should work with any approximate mode (find_replace, linear, early_fill, etc.) because it operates on the final set of placed tags.
- It could be a separate strategy (`GroupApproximateStrategy`) that internally uses gap-filling after sequential tag placement.
- Alternatively, make it a generic post-processing step in `ScheduleGenerator.finalize_day()` that runs whenever any gap filler tags are present, regardless of mode.

### 4. UI Integration
- Add a "Gap Filler" tag type in the tag management UI.
- When creating/editing a tag, user can mark it as a "Gap filler" and select a video collection, max duration, and options.
- In approximate mode dropdown, maybe a separate "Group + Gap" mode that combines the two behaviors, or simply treat Gap Tag as a tag like any other and the scheduler automatically handles it if present.

### 5. Configuration Choices
- Should gap fillers respect the tag's original `start_time`/`end_time`? Probably not; they are time-agnostic and fill any gap.
- If multiple gaps exist, should the filler try to distribute videos evenly or fill one gap completely before moving to the next? Likely fill sequentially.
- What about overnight gaps? If doing multi-day schedule, each day's gaps are filled independently.

### 6. Files to Modify
- `data_models.py`: Add `is_gap_filler` flag and properties to `Tag` class.
- `scheduler.py`: Implement `_fill_gaps()` method that processes all gap filler tags for a day.
- `strategies.py`: Ensure all approximate strategies call gap filling after tag placement (or integrate into `_approximate_finalize_day`).
- UI dialogs: Add gap filler options (e.g., in `RandomFillDialog` or new `GapDialog`).
- `daypart_scheduler.py`: Update tag list UI to show gap fillers, and ensure they're included in the tag manager.

### 7. Success Criteria
- All gaps between tags (and before first/after last) are filled with gap content, up to the maximum specified duration.
- No gap filler video overlaps any primary tag.
- The schedule remains continuous with no empty time slots (unless gap filler collection is exhausted).
- Multiple gap fillers cooperate via priority.
